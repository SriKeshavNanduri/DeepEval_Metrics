## Overview

This repository demonstrates how to evaluate an LLM answer using three evaluation metrics from `deepeval.metrics`:

- `AnswerRelevancyMetric`
- `FaithfulnessMetric`
- `ContextualPrecisionMetric`

The `main.py` script performs retrieval-augmented generation using a local text file (`rag_data.txt`) and OpenAI embeddings/chat completions, then evaluates the generated answer against a ground truth answer.

## What `main.py` Does

`main.py` runs the following sequence:

1. Load `rag_data.txt` and split it into overlapping text chunks.
2. Generate embeddings for each chunk with OpenAI embeddings.
3. Build a FAISS index over the chunk embeddings.
4. Run a sample query and retrieve the top 3 most relevant chunks.
5. Send the retrieved context plus the question to an OpenAI chat completion model.
6. Collect the LLM answer and compare it against a manual ground truth answer.
7. Create an `LLMTestCase` containing the input query, generated answer, expected answer, and retrieval context.
8. Instantiate the three metrics and evaluate them on the test case.
9. Print each metric's score and explanation.

## Metric Definitions

### `AnswerRelevancyMetric`

- Definition: Measures whether the model's answer is relevant to the original user query.
- What it means: A high relevancy score means the answer addresses the question content directly and does not drift off-topic.
- What it does not mean: It does not mean the answer is factually correct or supported by the retrieval context.
- Important example: If the query asks "When did Steve Jobs associate with Pixar?", an answer like "Steve Jobs never associated with Pixar until 2025" is still on-topic and may receive a high relevancy score even though it is wrong.
- Example: A good relevancy outcome is an answer that mentions Steve Jobs and Pixar dates rather than unrelated company history.

### `FaithfulnessMetric`

- Definition: Measures whether the model's answer is supported by the provided retrieval context and does not introduce unsupported or hallucinated facts.
- What it means: A faithful answer stays grounded in the retrieved evidence and avoids making claims not backed by the source text.
- What it does not mean: It does not guarantee the answer is complete or fully correct; it only checks if the answer is supported by the context.
- Important example: If the retrieved context contains only that Steve Jobs was associated with Pixar from 1986 to 2006, an answer that repeats this fact is faithful. But an answer that adds unverified claims like "Steve Jobs sold Pixar in 1999" would be unfaithful.
- Example: If the retrieved context only contains Steve Jobs and Pixar dates, the answer should not invent unrelated details like Steve Jobs leaving Pixar in a different year.

### `ContextualPrecisionMetric`

- Definition: Measures how precisely the answer uses the retrieved context to answer the query.
- What it means: A precise answer efficiently uses relevant context and avoids adding unnecessary or vague information.
- What it does not mean: It does not mean the answer is correct just because it is concise; it measures contextual focus, not factual verification.
- Important example: An answer that says "Steve Jobs worked with Pixar" is too vague and may score lower on precision, even though it is relevant. A higher precision answer would state the exact years and relationships found in the retrieved context.
- Example: A precise answer would answer the question with the specific Pixar association dates from the context rather than a broad summary of Steve Jobs' career.

## Example Evaluation Flow

In `main.py`, the evaluation is built around one sample test case:

- `input`: the user question string.
- `actual_output`: the LLM-generated answer from the chat completion.
- `expected_output`: the ground truth answer defined in the script.
- `retrieval_context`: the list of retrieved text chunks used to inform the answer.

Then the script creates the three metrics with `threshold=0.7`, runs `evaluate(...)`, and prints:

- `Metric`: metric name
- `Score`: numeric score for that metric
- `Reason`: the explanation generated by the metric

## How to Use

1. Ensure `OPENAI_API_KEY` is set in your environment or `.env` file.
2. Place your retrieval data in `rag_data.txt`.
3. Run:

```bash
python main.py
```

4. Review the printed metrics to understand answer relevancy, faithfulness, and contextual precision.
